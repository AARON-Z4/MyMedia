Base URL

Base URL 
http://localhost:5000
ℹ

All endpoints are relative to this base URL. For production deployments, replace localhost:5000 with your server's domain. The REACT_APP_API_URL environment variable configures the frontend to point at a different backend.

Environment Variables

Error Handling

All errors return a consistent JSON envelope

Error Response Shape 
{
  "error": "Human-readable message",
  "details": [/* only present on validation errors */
    {
      "type": "field",
      "msg": "Name is required",
      "path": "name",
      "location": "body"
    }
  ]
}
StatusMeaningReturned When
200OKSuccessful GET, DELETE, or PUT
201CreatedResource successfully created (POST)
400Bad RequestValidation failed or invalid ID format
404Not FoundResource with given ID does not exist
409ConflictDuplicate entry (unique constraint violation)
500Server ErrorUnexpected server-side error

pageinteger1—Page number (min: 1)

{
  "data": [ /* array of resources */ ],

    "page":  1,

Projects

Create and manage top-level projects

POST /projects Create a new project

Request Body · application/json

FieldTypeRequiredDescription
name string required Project name. Cannot be empty or whitespace.
description string optional Short description of the project. Defaults to "".
Request 
POST /projects
Content-Type: application/json

{
  "name": "Website Redesign",

}

  "__v": 0

  "error": "Validation failed",
  "details": [

GET /projects/:project_id/tasks List tasks — filter, sort, paginate
✓

All three query params can be combined in a single request: ?status=todo&sort=due_date&page=1&limit=5

Path Parameters

ParamTypeDescription
:project_idMongoDB ObjectIdParent project's _id

Query Parameters

ParamTypeDefaultDescription
statusenum— Filter tasks by status: todo · in-progress · done
sortstring— Pass due_date to sort ascending by due date (earliest first). Omit for default: newest first.
pageinteger1 Page number
limitinteger20 Items per page (max: 100)
Example Requests 
# All tasks
GET /projects/664f.../tasks

# Filter by status
GET /projects/664f.../tasks?status=in-progress

# Sort by due date (ascending)
GET /projects/664f.../tasks?sort=due_date

# Filter + sort + paginate
GET /projects/664f.../tasks?status=todo&sort=due_date&page=1&limit=5
200OK
{

    {

Data Schemas

MongoDB document shapes returned by all endpoints

Project Object

Project Schema 
{
  "_id":         "MongoDB ObjectId (string)",
  "name":        "string — required, trimmed",

Task Object

Task Schema 
{
  "_id":         "MongoDB ObjectId (string)",
  "project_id":  "MongoDB ObjectId — ref to parent Project",
  "title":       "string — required, trimmed",
  "description": "string — optional, default: \"\"",
  "status":      "enum: \"todo\" | \"in-progress\" | \"done\"",      // default: "todo"
  "priority":    "enum: \"low\" | \"medium\" | \"high\"",          // default: "medium"
  "due_date":    "ISO 8601 datetime | null — optional",
  "created_at":  "ISO 8601 datetime — auto-set on creation"
}
